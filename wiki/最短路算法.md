  - 顶点标号法# 最短路算法

> **问题背景**：许多最优化问题都可转化为在一个赋权图中找出某类具有最小（或最大）权的子图，其中之一就是最短路问题：给出一个连接各城镇的铁路网络，在这个网络的两个指定城镇之间试确定一条最短路线。数学模型为：在一个赋权图中的两个指定顶点 $a$ 和 $b$ 之间找出一条最短 $(a, b)$ 路。

1959 年，Dijkstra 发表了一篇名为 "A Note on Two Problems in Connexion with Graphs"，这篇文章给出了解决最短路问题的著名的 Dijkstra 算法。

## 算法思想

Dijkstra 算法（顶点标号法）不仅找到了最短的 $(a, b)$ 路，而且给出了 $a$ 到图 $G$ 的所有其他顶点的最短路。

![[file-20260422204922596-Dijkstra记号.png]]

**记号说明**：
- $l(e)$：边 $e$ 的权重
- $t(a_k)$：点 $a_k$ 的标号值，表示点 $a_1 = a$ 到 $a_k$ 的最短路长度
- $A_i = \{a_1, a_2, \ldots, a_i\}$：已经标号的顶点集合
- $T_i$：$a_1$ 到 $a_i$ 的最短路上的边所在的集合
- $N(a_k)$：与点 $a_k$ 相邻的所有点的集合，称为 $a_k$ 的邻集

## 算法步骤

![[file-20260422204922592-Dijkstra算法步骤.png]]

**Dijkstra 算法**：
1. 记 $a_1 = a$，$t(a_1) = 0$，$A_1 = \{a_1\}$，$T_1 = \emptyset$；
2. 若已经得到 $A_i = \{a_1, a_2, \ldots, a_i\}$ 且对于 $1 \leq k \leq i$，$t(a_k)$ 已知。对每一个 $a_k \in A_i$，求一点：
   $$b_k^{(i)} \in N(a_k) - A_i = B_k^{(i)}$$
   使得：$l(a_kb_k^{(i)}) = \min_{v \in B_k^{(i)}} l(a_kv)$；
3. 设有 $m_i$，$1 \leq m_i \leq i$，而 $b_{m_i}^{(i)}$ 使 $t(a_{m_i}) + l(a_{m_i}b_{m_i}^{(i)})$ 取最小值。令：
   $$a_{i+1} = b_{m_i}^{(i)}, \quad t(a_{i+1}) = t(a_{m_i}) + l(a_{m_i}a_{i+1}), \quad T_{i+1} = T_i \cup \{a_{m_i}a_{i+1}\}$$；
4. 若 $a_{i+1} = b$，停止；否则，记 $A_{i+1} = A_i \cup \{a_{i+1}\}$，转 (2)。

## 例子

![[file-20260422204922610-Dijkstra例子.png]]

如图所示，求点 $a$ 到点 $b$ 的距离。

解：
1. $A_1 = \{a\}$，$t(a) = 0$，$T_1 = \emptyset$；
2. $b_1^{(1)} = v_3$；
3. $m_1 = 1$，$a_2 = v_3$，$t(v_3) = t(a) + l(av_3) = 1$（最小），$T_2 = \{av_3\}$；
4. $A_2 = \{a, v_3\}$，$b_1^{(2)} = v_1$，$b_2^{(2)} = v_2$；
5. $m_2 = 1$，$a_3 = v_1$，$t(v_1) = t(a) + l(av_1) = 2$（最小），$T_3 = \{av_3, av_1\}$；
6. $A_3 = \{a, v_3, v_1\}$，$b_1^{(3)} = v_2$，$b_2^{(3)} = v_2$，$b_3^{(3)} = v_4$；
7. $m_3 = 3$，$a_4 = v_4$，$t(v_4) = t(v_1) + l(v_1v_4) = 3$（最小），$T_4 = \{av_3, av_1, v_1v_4\}$；

...（继续迭代直到到达 $b$）

最终：$a$ 与 $b$ 的距离为 $d(a, b) = 11$，最短路为 $a \to v_1 \to v_4 \to v_2 \to v_6 \to b$。

## 时间复杂度

Dijkstra 算法的时间复杂度为 $O(n^2)$，其中 $n$ 为顶点数。使用优先队列（堆）优化后可达到 $O(m \log n)$。

## 适用场景与优缺点

- **适用**：边权为非负数的赋权图
- **优点**：能找到源点到所有其他顶点的最短路
- **缺点**：不能处理负权边
