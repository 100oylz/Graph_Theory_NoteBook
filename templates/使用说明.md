# 模板使用说明

## 安装配置

1. 安装 **Templater** 或 **Templates** 插件（Obsidian 官方模板或第三方 Templater）
2. 在插件设置中，将模板文件夹路径设为 `templates/`

## 使用方法

### 方法1：使用 Templates 插件（官方）
1. 按 `Ctrl+P` 打开命令面板
2. 输入 `Templates: Insert template`
3. 选择对应的模板类型（概念/算法/定理/问题）

### 方法2：使用 Templater 插件（推荐）
1. 按 `Ctrl+P` 打开命令面板
2. 输入 `Templater: Open Insert Template Modal`
3. 选择模板

### 方法3：新建文件时自动应用
- 在 Templater 设置中，可以为特定文件夹配置自动模板
- 例如：`wiki/` 文件夹新建文件时自动使用 `通用` 模板

## 模板变量

当前模板支持以下占位符（需配合 Templater 插件）：

| 占位符 | 含义 |
|--------|------|
| `{{title}}` | 当前文件标题（文件名） |
| `{{date}}` | 当前日期（YYYY-MM-DD） |
| `{{time}}` | 当前时间（HH:mm） |

## YAML 字段说明

| 字段 | 用途 | 填写示例 |
|------|------|----------|
| `chapter` | Dataview 查询用章节名（纯字符串） | `第1章 图的基本概念` |
| `parent` | Breadcrumbs 面包屑导航用（wikilink） | `[[第1章 图的基本概念]]` |
| `category` | 知识类型（概念/算法/定理/问题） | `概念` |
| `tags` | Obsidian 标签 | `[图论, 算法]` |
| `related` | 弱关联概念列表 | `[最短路, Dijkstra]` |
| `aliases` | 页面别名 | `[最短路径]` |

> **注意**：`chapter` 和 `parent` 需要同时填写，前者用于 Dataview 筛选，后者用于 Breadcrumbs 层级导航。两者指向同一个章节页面，但格式不同。

## 模板类型对照

| 模板 | 适用场景 | category |
|------|----------|----------|
| `概念.md` | 定义、性质类的知识点 | 概念 |
| `算法.md` | 算法描述、伪代码、复杂度 | 算法 |
| `定理.md` | 定理陈述与证明 | 定理 |
| `问题.md` | 经典问题描述与解法 | 问题 |
| `通用.md` | 不确定类别时使用 | 任意 |

## 快速创建流程

```
1. 在 wiki/ 目录下新建文件（文件名即概念名）
2. 使用模板插入对应模板
3. 填写 chapter 字段（如：第1章 图的基本概念）
4. 补充 definition / 算法步骤 / 证明等内容
5. 在相关概念区添加 [[wikilink]] 建立连接
```

## 注意事项

- **文件名** = 概念名，后续被引用时直接显示该名称
- **chapter** 字段请填写完整章名（纯字符串），如 `第1章 图的基本概念`，用于 Dataview 查询
- **parent** 字段填写 wikilink 格式，如 `[[第1章 图的基本概念]]`，用于 Breadcrumbs 面包屑导航
- **related** 字段可填写关联概念的文件名，用于后续关系图谱分析
